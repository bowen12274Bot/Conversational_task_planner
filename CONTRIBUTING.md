# Contributing Guide

## 目的

本文件用於說明成員應如何參與此專案的開發與協作。

## 工作流程

1. 先接收被指派的任務
2. 確認該任務關聯的架構層、模組與規格文件
3. 將任務再拆解為自己可執行的小任務
4. 依情況自行完成，或整理後交由 AI 協助產出
5. 完成後整理結果，確認是否需要同步更新文件或規格

## 文件分工

目前文件分工如下：

- `docs/project_overview.md`
  - 專案總覽文件
  - 用於了解專案背景、問題定義、專案目標、主要使用情境與預期成果

- `docs/architecture.md`
  - 系統架構文件
  - 用於了解目前的架構分層與層間互動

- `docs/module_description.md`
  - 系統模組說明
  - 用於了解目前的核心模組與各模組職責

- `docs/development_plan/`
  - 開發規劃文件資料夾
  - 用於了解目前團隊的計畫與工作範圍

- `docs/specs/`
  - 共用規格資料夾
  - 用於放置 API 規格、資料契約、畫面需求與資料實體草稿等正式規格

## 私人資料夾

專案根目錄中的 `.private/` 為個人使用的私人工作資料夾，不納入版控，可用於存放：

- 個人任務文件
- 工作草稿
- AI 指派文件
- 暫存筆記

此資料夾主要用於個人工作過程中的中介內容，不作為正式規格或正式專案文件的存放位置，不要求一定要使用。

由於專案開發過程中可能會搭配 AI 工具協助處理任務、整理草稿或準備工作文件，因此私人資料夾可讓個人工作與 AI 對話有較高的彈性，可善加利用。

若本機尚未建立 `.private/`，可於專案根目錄執行以下指令快速建立：

```powershell
npm run init:private
```

若需要重建檔案，請手動刪除 `.private/` 資料夾，然後再執行一次建立指令。

## 分支使用

目前專案採任務指派型協作方式，因此收到任務後，建議先建立對應的工作分支，再於該分支上進行開發與整理。

- `main` 主分支
  - 保持可閱讀、可整理、可 demo 的狀態
  - 強烈建議不要直接將修改 push 到 `main`

- 創建分支
  - 每項任務建議使用對應分支處理
  - 若任務同時涵蓋多個功能，建議拆成多個分支，但不強制要求
  - 若不確定是否需要拆分，可先以一個分支處理，但應避免在同一分支混入過多不相關修改

- 分支命名
  - 建議依修改類型命名，例如 `feature/...`、`docs/...`、`fix/...`
  - 分支名稱可再根據實際工作內容補上主題，例如：
    - `docs/mvp-planning`
    - `feature/task-panel`
    - `feature/chat-flow`
    - `fix/api-format`
  - 當不知如何命名時，可以向 AI 請教命名建議，但要注意命名格式是否錯誤

- 合併方式
  - 平常採自己 `PR + merge` 方式處理
  - 若對自己的程式內容、修改影響或整合結果不夠確定，建議先請他人協助審核
  - 重要改動、較大範圍修改或涉及共用規格時，應優先請人協助確認後再合併
  - 推送或合併後，請主動通知其他成員目前更新內容，避免協作資訊落差

## 提交建議

提交時，請盡量讓每一筆提交都對應明確的工作成果，避免將過多不相關的內容混在一起。

- 提交原則
  - 一次提交盡量對應一項明確工作成果
  - 不要把多個無關任務混在同一筆提交
  - 以上規定不強制要求

- 訊息命名
  - 建議依修改類型命名，例如 `feat: ...`、`docs: ...`、`fix: ...`
  - 訊息內容可再根據實際工作內容補上主題
  - 當不知如何命名時，可以向 AI 請教命名建議，但要注意命名格式是否錯誤

- 示範案例
  - 建議例子
    - `docs: refine mvp milestones`
      - 只處理 MVP 里程碑規劃調整
    - `feat: add frontend shell layout`
      - 只處理前端基本畫面骨架
    - `fix: adjust init private script`
      - 只處理 `.private/` 初始化腳本修正

  - 不建議例子
    - `feat: update frontend and docs and api`
      - 同時混入前端、文件與 API 修改，範圍過大且不清楚
    - `misc: update project`
      - 無法看出實際修改內容與影響範圍
