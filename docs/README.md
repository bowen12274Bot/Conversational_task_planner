# Docs README

本資料夾用於整理專案的文件內容，依照文件用途區分為專案總覽、模組說明、架構設計、系統設計與開發規劃。

## 文件說明

- [project_overview.md](./project_overview.md)
  - 專案總覽文件
  - 包含專案背景、問題定義、專案目標、主要使用情境與預期成果

- [module_description.md](./module_description.md)
  - 系統模組說明文件
  - 包含 Context Engineering、Response、Questioning、Planning、Output Structuring 與 Persistence 等核心模組說明

- [system_design.md](./system_design.md)
  - 系統設計文件
  - 包含平台形式、技術架構、技術選型、資料設計方向與 AI 串接設計

- [architecture.md](./architecture.md)
  - 系統架構文件
  - 包含專案目前的整體架構分層、各層角色與層間互動流程

- [quick_start.md](./quick_start.md)
  - 快速啟動文件
  - 包含前端、後端與資料庫初始化的基本指令

- [development_plan/mvp_plan.md](./development_plan/mvp_plan.md)
  - MVP 開發規劃文件
  - 包含目前 MVP 的目標、範圍與階段重點

## 建議閱讀順序

1. `project_overview.md`
2. `quick_start.md`
3. `development_plan/mvp_plan.md`
4. `architecture.md`
5. `module_description.md`

## 備註

目前文件已依專案定位、模組設計、架構分層、系統設計與 MVP 規劃分開整理。後續若再擴充完整版本，建議持續維持文件分工清楚，避免不同文件承擔過多重複內容。
