# AI 串接方向設計

更新日期時間：2026-04-20 14:00:00

本文件用於整理本專案目前對 AI 串接方式的設計方向與技術構想，作為後續形成正式規格前的設計參考。

---

## 文件目的

本文件的重點在於整理 AI 串接的可能方向與目前可保留的技術構想，避免過早定死實作方案，同時保留後續收斂空間。

---

## 目前設計原則

### 一、AI 輸出需能被系統承接

不論輸出為 JSON 結果或文字結果，都應能被後端模組明確承接，而不是讓 AI 產生過度自由、難以接續處理的內容。

### 二、先不綁定特定供應商設計

現階段先不展開特定 AI 供應商的 API 細節，而優先整理本專案自己的 AI 使用方式與設計邊界。

### 三、串接方式應服務於模組需求

AI 串接方式應依模組所需輸出類型決定，而不是先由特定技術方案反推模組設計。

---

## 技術實作方向

目前可先預期，系統在技術上可能會透過預先定義的工具、技能或程式輔助方式，約束 AI 的輸入理解與輸出格式。

可能的實作方向包括：

- 以程式工具約束固定格式輸出
- 以 skill 輔助 AI 閱讀與使用特定程式工具
- 以既定流程承接 AI 回傳內容

本階段先保留這些方向，不先定義具體技術方案。

---

## 現階段先不處理的內容

以下內容現階段先不展開：

- 特定 AI 供應商 API 規格
- Prompt 細節設計
- 完整 function calling 或 schema binding 細節
- AI 成本、配額與部署策略
- 向量資料庫或長期記憶設計

---

## 後續可延伸方向

後續若專案進一步收斂，可再從本文件延伸出更正式的規格文件，例如：

- AI 串接規格
- 模組對 AI 的輸入輸出格式
- Prompt 設計原則
- 結構化輸出約束方式
