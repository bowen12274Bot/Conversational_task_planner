# 專案概述

更新日期時間：2026-04-16 21:55:00

本專案旨在設計一套結合對話式 AI 與互動式介面的任務規劃系統，聚焦於「作業或小專題截止前規劃」的使用情境。專案希望協助使用者從模糊目標出發，逐步形成較清楚的初步規劃方向，降低面對任務時的不知如何開始與安排的壓力。

# 專案背景

學生在面對作業或小專題時，常見的困難並非單純執行，而是不知道如何開始規劃。特別是在接近截止時間時，使用者通常只掌握模糊目標，例如「要完成作業」或「要做完專題」，卻難以立即拆解為具體步驟，也不容易估計每個部分所需時間。

在這類情境下，使用者需要的往往不是單純的提醒工具，而是一種能協助整理需求、釐清方向並建立初步規劃的支援方式。因此，本專案以學生在截止前的規劃需求作為出發點，希望建立一套更貼近真實使用情境的任務規劃系統。

# 問題定義

目前學生在面對作業或小專題規劃時，常見問題主要包括以下幾點：

- 不知道如何開始，難以將模糊目標轉化為可執行的起點
- 任務拆解困難，無法判斷大任務應如何分成較小步驟
- 時間估計不易，難以掌握每個部分大約需要多少投入
- 現有工具多偏向記錄待辦事項或提供文字建議，缺乏同時兼具引導與結構化規劃的方式

因此，本專案希望處理的核心問題，是如何在模糊需求與具體規劃之間建立一個有效的過渡機制，幫助使用者更順利地展開任務。

# 專案目標

本專案的主要目標如下：

## 1. 協助使用者從模糊目標出發

使用者不需要一開始就提出完整計畫，而是能先以自然語言描述想完成的作業或小專題，作為後續規劃的起點。

## 2. 協助形成初步可執行的規劃方向

系統希望協助使用者將大致的任務方向逐步整理成較具體的規劃內容，使使用者能更容易理解自己應先從哪裡開始。

## 3. 降低開始行動的門檻

本專案希望減少使用者在面對複雜任務時的混亂感與拖延感，讓規劃結果能成為實際開始行動的基礎，而不只是停留在抽象想法。

## 4. 提供比純文字建議更清楚的規劃成果

除了提供規劃方向外，系統也希望讓結果更具結構性，使使用者能更清楚掌握任務內容、優先順序與初步安排。

# 主要使用情境

本專案目前聚焦的使用情境具有以下特徵：

- 有明確截止時間
- 任務通常不只一個步驟
- 使用者知道需要完成什麼，但不清楚如何拆解與安排

因此，目前最主要的使用情境包括：

- 作業截止前規劃
- 小專題截止前規劃
- 時間有限情況下的初步任務安排

# 預期成果

本專案預期完成一套可作為任務規劃原型的系統，協助學生在面對作業或小專題時，能從模糊需求逐步形成較清楚的規劃方向。

此成果的重點不在於取代完整專案管理工具，而是在於驗證一種更適合學生情境的互動式規劃方式，並作為後續持續發展與優化的基礎。
