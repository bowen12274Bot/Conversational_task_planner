# AI 文字結果輸出 API 規格

## 文件名稱

- `ai_text_output_api`

## API 端點

- 內部串接規格，端點形式待後續實作時定義

## API 目的

- 提供內部模組經由 `AI service layer` 呼叫外部 AI
- 根據既有分析結果生成最終文字內容
- 供文字輸出相關流程承接

## 適用範圍

- `Response Module`

## 分組方式

- 本 API 已依文字結果輸出用途獨立成一支 API
- 在同一支 API 內，仍可依不同文字輸出目的再做分組
- 不同分組可有不同的輸入格式、輸出格式與處理要求

目前可預期的分組包括：

- `questioning_response`

### 分組說明

- `questioning_response`
  - 用於根據 `Questioning Module` 的判斷結果，生成提供給使用者閱讀的追問或說明文字。

## 輸入內容

- 依分組需求提供對應輸入資料
- 文字輸出目標

## 成功條件

- `AI service layer` 成功完成呼叫

## 回應訊息

- 成功時，回傳文字結果
- 失敗時，回傳錯誤狀態與錯誤資訊

## 狀態碼規範

| 狀態碼 | 名稱 | 說明 |
|---|---|---|
| `200` | `OK` | 本次文字結果輸出請求成功完成 |
| `500` | `Internal Server Error` | AI 串接或文字生成處理過程發生錯誤 |

## 錯誤訊息格式

在錯誤情境下，回應中至少應包含：

- `error_message`
- `error_stage`
