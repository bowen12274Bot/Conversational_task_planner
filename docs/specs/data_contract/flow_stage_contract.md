# Flow Stage 資料契約

本文件用於定義 `flow_stage` 在資料傳輸情境下所需的最小資料內容，作為後端流程控制層讀取、傳遞與引用流程節點時的共用基準。

---

## 文件目的

本文件的重點，不在於描述流程圖如何閱讀，也不在於重述節點命名規則，而是在於收斂：

- 流程節點在資料傳輸時真正需要哪些欄位
- 流程節點之間的前後關係應如何表示
- 哪些欄位屬於執行時必要資料

---

## 資料物件一：Flow Stage 資料

### 定義

用於表示單一 `flow_stage` 節點在資料傳輸情境下所需內容的結構化資料物件。

### 組成內容

- 系統識別
- 處理單位
- 上一步
- 下一步

### 作用

- 提供流程控制層穩定引用單一節點的方式
- 提供流程控制層判斷該節點應交由哪一個處理單位執行的依據
- 提供流程控制層讀取節點前後關係時的共用格式

### 特性

- 此資料物件只保留資料流動中必要的核心內容
- `系統識別` 為節點在系統中的穩定標的
- `處理單位` 用於指出該節點所對應的主要架構層或模組層處理單位
- `上一步` 與 `下一步` 可對應單一節點、多個節點，或流程開始 / 結束狀態
- 當流程涉及擇一路線、同時傳送或結合時，仍以 `上一步` 與 `下一步` 的多值關係表達
- `代號`、`節點名稱`、`說明` 屬於文件與辨識用途，不列入本資料契約的核心傳輸內容

### 名詞定義

- `系統識別`
  - 用於在系統內部穩定指向單一流程節點的識別欄位。

- `處理單位`
  - 用於表示本節點主要涉及的架構層或模組層處理單位，以及資料在該節點中的主要流向。

- `上一步`
  - 用於表示目前節點的前一個節點、前一組節點，或流程開始狀態。

- `下一步`
  - 用於表示目前節點的下一個節點、下一組節點，或流程結束狀態。

## 資料物件二：Flow Stage 執行狀態

### 定義

用於表示單一 `flow_stage` 節點在流程執行期間之狀態資訊的結構化資料物件。

### 組成內容

- 目前節點系統識別
- 步驟狀態
- 流程狀態
- 單節點重試次數
- 累積錯誤次數
- 錯誤層深度
- 最近一次錯誤資訊

### 作用

- 提供後端流程控制層在執行期間判斷目前節點狀態的依據
- 提供後端流程控制層進行重試、回退或中止判斷時的共用格式
- 提供流程執行中錯誤處理與狀態追蹤的最小資料基準

### 特性

- 此資料物件屬於執行時資料，而非靜態流程定義資料
- `步驟狀態` 用於表示目前節點這一次執行的結果
- `流程狀態` 用於表示整體流程目前是否仍在進行、已完成或已失敗
- 錯誤次數與錯誤層深度用於限制重試與回退範圍

### 名詞定義

- `目前節點系統識別`
  - 用於指出目前正在執行或判斷中的流程節點。

- `步驟狀態`
  - 用於表示目前節點這一次執行的結果，例如完成、重試同一節點、回退至其他節點或中止。

- `流程狀態`
  - 用於表示整體流程目前的執行狀態，例如執行中、已完成或已失敗。

- `單節點重試次數`
  - 用於表示目前節點已累積的重試次數。

- `累積錯誤次數`
  - 用於表示本輪流程從開始至目前為止累積發生的錯誤次數。

- `錯誤層深度`
  - 用於表示目前流程已進入的錯誤回退層級深度。

- `最近一次錯誤資訊`
  - 用於表示目前流程最近一次錯誤的簡短描述與必要辨識資訊。

## 資料物件關係

| 階段 | 資料物件之間的關係 | 說明 |
|---|---|---|
| 1 | Flow Stage 資料 -> [流程階段參考](./flow_stage_reference.md) | `flow_stage_reference.md` 中的流程關係圖，可視為 `Flow Stage 資料` 在文件中的整合呈現 |
| 2 | Flow Stage 資料 + Flow Stage 執行狀態 -> 後端流程控制層 | 後端流程控制層可依 `Flow Stage 資料` 取得節點定義，並依 `Flow Stage 執行狀態` 判斷目前執行結果與錯誤處理方式 |

## 與流程階段參考的對應

[流程階段參考](./flow_stage_reference.md) 中的流程關係圖，可理解為：

- `Flow Stage 資料`
  - 對應表格中的：
    - 系統識別
    - 處理單位
    - 上一步
    - 下一步

表格中的 `代號`、`節點名稱` 與 `說明`，可作為文件閱讀與人工辨識使用，但不屬於本文件所定義的核心傳輸資料。

`Flow Stage 執行狀態` 不直接對應 [流程階段參考](./flow_stage_reference.md) 的靜態關係表格，而是用於流程實際執行時，補充目前節點所處的執行狀態與錯誤狀態。
