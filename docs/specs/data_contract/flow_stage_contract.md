# Flow Stage 資料契約

本文件用於定義 `flow_stage` 在資料傳輸情境下所需的最小資料內容，作為後端流程控制層讀取、傳遞與引用流程節點時的共用基準。

---

## 文件目的

本文件的重點，不在於描述流程圖如何閱讀，也不在於重述節點命名規則，而是在於收斂：

- 流程節點在資料傳輸時真正需要哪些欄位
- 流程節點之間的前後關係應如何表示
- 哪些欄位屬於執行時必要資料

---

## 資料物件一：Flow Stage 資料

### 定義

用於表示單一 `flow_stage` 節點在資料傳輸情境下所需內容的結構化資料物件。

### 組成內容

- 系統識別
- 處理單位
- 上一步
- 下一步

### 作用

- 提供流程控制層穩定引用單一節點的方式
- 提供流程控制層判斷該節點應交由哪一個處理單位執行的依據
- 提供流程控制層讀取節點前後關係時的共用格式

### 特性

- 此資料物件只保留資料流動中必要的核心內容
- `系統識別` 為節點在系統中的穩定標的
- `處理單位` 用於指出該節點所對應的主要架構層或模組層處理單位
- `上一步` 與 `下一步` 可對應單一節點、多個節點，或流程開始 / 結束狀態
- 當流程涉及擇一路線、同時傳送或結合時，仍以 `上一步` 與 `下一步` 的多值關係表達
- `代號`、`節點名稱`、`說明` 屬於文件與辨識用途，不列入本資料契約的核心傳輸內容

### 名詞定義

- `系統識別`
  - 用於在系統內部穩定指向單一流程節點的識別欄位。

- `處理單位`
  - 用於表示本節點主要涉及的架構層或模組層處理單位，以及資料在該節點中的主要流向。

- `上一步`
  - 用於表示目前節點的前一個節點、前一組節點，或流程開始狀態。

- `下一步`
  - 用於表示目前節點的下一個節點、下一組節點，或流程結束狀態。

## 資料物件二：Flow Stage 執行狀態

### 定義

用於表示單一 `flow_stage` 節點在流程執行期間之狀態資訊的結構化資料物件。

### 組成內容

- 目前節點系統識別
- 步驟狀態
- 單節點重試次數

### 作用

- 提供後端流程控制層在執行期間判斷目前節點狀態的依據
- 提供後端流程控制層進行單節點重試、回退或中止判斷時的共用格式
- 提供流程執行中錯誤處理與狀態追蹤的最小資料基準

### 特性

- 此資料物件屬於執行時資料，而非靜態流程定義資料
- `步驟狀態` 用於表示目前節點這一次執行的結果
- `單節點重試次數` 用於限制目前節點的重試範圍

### 名詞定義

- `目前節點系統識別`
  - 用於指出目前正在執行或判斷中的流程節點。

- `步驟狀態`
  - 用於表示目前節點這一次執行的結果，例如完成、重試同一節點、回退至其他節點或中止。
  - 可用值建議為：
    - `done`：目前節點本次執行完成，可交由控制層判斷下一步。
    - `retry_same_stage`：目前節點本次執行失敗，應於同一節點重試。
    - `fallback_to_stage`：目前節點本次執行失敗，應回退至其他節點。
    - `abort`：目前節點本次執行應中止，且中止狀態需由控制層向外層傳播。

- `單節點重試次數`
  - 用於表示目前節點已累積的重試次數。

## 資料物件三：Flow Run 狀態

### 定義

用於表示整體流程在執行期間之狀態資訊的結構化資料物件。

### 組成內容

- 流程狀態
- 累積錯誤次數
- 已經過歷史

### 作用

- 提供後端流程控制層判斷整體流程目前是否仍可持續推進的依據
- 提供後端流程控制層判斷本輪流程是否應完成、失敗或中止的共用格式
- 提供流程執行中已實際走過之節點軌跡，作為除錯與回溯依據

### 特性

- 此資料物件屬於整體流程層級的執行時資料
- `流程狀態` 用於表示整體流程目前是否仍在進行、已完成或已失敗
- `累積錯誤次數` 用於限制整體流程層級的錯誤擴張與回退範圍
- `已經過歷史` 用於補充本輪流程在實際執行時已走過的整體路徑脈絡

### 名詞定義

- `流程狀態`
  - 用於表示整體流程目前的執行狀態，例如執行中、已完成或已失敗。
  - 可用值建議為：
    - `running`：整體流程仍在執行中。
    - `completed`：整體流程已完成並可結束。
    - `failed`：整體流程已失敗，無法再持續推進。

- `累積錯誤次數`
  - 用於表示本輪流程從開始至目前為止累積發生的錯誤次數。

- `已經過歷史`
  - 用於表示本輪流程目前為止實際經過的節點紀錄，可作為除錯、回溯與執行路徑辨識使用。

## 資料物件四：錯誤鏈資料

### 定義

用於表示單一錯誤鏈在流程執行期間之進入、退出、層級與父子鏈關係的結構化資料物件。

### 組成內容

- 錯誤鏈識別
- 進入點系統識別
- 退出點系統識別
- 錯誤深度
- 父錯誤鏈識別
- 錯誤原因

### 作用

- 提供後端流程控制層記錄單一錯誤鏈從何處進入與從何處退出的依據
- 提供後端流程控制層表達多層錯誤鏈父子關係的共用格式
- 提供後端流程控制層在錯誤回退、重作與中止判斷時的追蹤資料

### 特性

- 此資料物件屬於整體流程層級的錯誤追蹤資料
- `進入點系統識別` 與 `退出點系統識別` 用於標示錯誤鏈的起訖位置
- `錯誤深度` 用於標示本錯誤鏈所在的錯誤層級
- `父錯誤鏈識別` 用於表達本錯誤鏈是否由上一層錯誤鏈衍生

### 名詞定義

- `錯誤鏈識別`
  - 用於穩定指向單一錯誤鏈資料的識別欄位。

- `進入點系統識別`
  - 用於表示本錯誤鏈最初進入錯誤處理流程時所對應的流程節點系統識別。

- `退出點系統識別`
  - 用於表示本錯誤鏈完成退出、收斂或中止時所對應的流程節點系統識別。

- `錯誤深度`
  - 用於表示本錯誤鏈在整體流程中的錯誤層級深度。

- `父錯誤鏈識別`
  - 用於表示本錯誤鏈所屬的上一層錯誤鏈識別；若無父層則可為空值。

- `錯誤原因`
  - 用於表示本錯誤鏈被建立時所對應的主要錯誤原因描述。

## 資料物件五：錯誤鏈堆疊

### 定義

用於表示本輪流程在執行期間目前作用中之多層錯誤鏈資料堆疊的結構化資料物件。

### 組成內容

- 錯誤鏈資料清單

### 作用

- 提供後端流程控制層維護多層錯誤鏈 push / pop 的共用格式
- 提供後端流程控制層進行子錯誤進入、退出與回退控制時的依據

### 特性

- 此資料物件屬於整體流程層級的執行時堆疊資料
- 內部各筆紀錄應對應單一 `錯誤鏈資料`
- 可同時承載主錯誤鏈與其子錯誤鏈的巢狀作用結構
- 當錯誤鏈退出時，可依控制層邏輯將對應資料自堆疊中移除
- 若任一內層錯誤鏈狀態進入中止，該中止狀態應由控制層逐層向外更新至整體流程層

### 名詞定義

- `錯誤鏈資料清單`
  - 用於表示本輪流程目前仍作用中之多筆錯誤鏈資料堆疊集合。

## 資料物件關係

| 階段 | 資料物件之間的關係 | 說明 |
|---|---|---|
| 1 | Flow Stage 資料 -> [流程階段參考](./flow_stage_reference.md) | `flow_stage_reference.md` 中的流程關係圖，可視為 `Flow Stage 資料` 在文件中的整合呈現 |
| 2 | Flow Stage 資料 + Flow Stage 執行狀態 -> 後端流程控制層 | 後端流程控制層可依 `Flow Stage 資料` 取得節點定義，並依 `Flow Stage 執行狀態` 判斷目前節點的執行結果與單節點錯誤處理方式 |
| 3 | Flow Run 狀態 + 錯誤鏈資料 -> 後端流程控制層 | 後端流程控制層可依整體流程層狀態與目前錯誤鏈資料，判斷流程是否應進入、退出、重作或中止錯誤處理 |
| 4 | 錯誤鏈資料 + 錯誤鏈堆疊 -> 後端流程控制層 | 後端流程控制層可依目前錯誤鏈與錯誤鏈堆疊資料，追蹤多層錯誤結構與父子鏈關係 |

---

## 與流程階段參考的對應

[流程階段參考](./flow_stage_reference.md) 中的流程關係圖，可理解為：

- `Flow Stage 資料`
  - 對應表格中的：
    - 系統識別
    - 處理單位
    - 上一步
    - 下一步

表格中的 `代號`、`節點名稱` 與 `說明`，可作為文件閱讀與人工辨識使用，但不屬於本文件所定義的核心傳輸資料。

`Flow Stage 執行狀態` 不直接對應 [流程階段參考](./flow_stage_reference.md) 的靜態關係表格，而是用於流程實際執行時，補充目前節點所處的執行狀態與錯誤狀態。

`Flow Run 狀態`、`錯誤鏈資料` 與 `錯誤鏈堆疊` 亦不直接對應 [流程階段參考](./flow_stage_reference.md) 的靜態關係表格，而是用於補充整體流程與多層錯誤鏈在執行期間的動態狀態。
