# 持久化至控制層的回傳對話紀錄歷史資料契約

本文件用於定義 `Persistence Module` 將對話紀錄歷史資料回傳至後端控制層時的核心資料物件。

---

## 資料物件一：歷史對話紀錄資料

### 定義

由 `Persistence Module` 自持久化資料中取出、整理並打包完成後，回傳給後端控制層的歷史對話紀錄資料。

### 組成內容

- 歷史對話 JSON

### 作用

- 讓控制層取得指定對話的歷史紀錄內容
- 提供控制層後續回傳前端歷史查詢結果所需的基礎資料

### 特性

- 此資料物件位於架構層，不展開資料庫表結構細節
- 可假設 `Persistence Module` 已完成資料查詢、整理與打包
- 現階段先維持在支援歷史對話紀錄回傳的最小範圍

### 名詞定義

- 歷史對話 JSON
  - 指由 `Persistence Module` 整理並打包完成的歷史對話資料，其內容包含該對話的回合資料與各回合對應的訊息資料。

其最小資料規格如下：

- `turns`
  - 歷史回合資料陣列。

- `turn`
  - `turns` 陣列中的單一回合物件，其內容至少包含 `turn_id` 與 `messages`。

- `messages`
  - 屬於單一 `turn` 的訊息資料陣列。

- `message`
  - `messages` 陣列中的單一訊息物件，其內容至少包含 `type` 與 `content`。

### 資料示範

```json
{
  "turns": [
    {
      "turn_id": "turn_001",
      "messages": [
        {
          "type": "user",
          "content": "我下週要交資料庫作業，但我還不知道怎麼開始。"
        },
        {
          "type": "ai",
          "content": "你目前已經知道作業題目和截止時間了嗎？"
        }
      ]
    },
    {
      "turn_id": "turn_002",
      "messages": [
        {
          "type": "user",
          "content": "題目是做一個簡單的選課系統，截止時間是下週五。"
        },
        {
          "type": "ai",
          "content": "目前看起來還需要先確認你要做到哪些功能，例如登入、選課、退選與查詢課表。"
        }
      ]
    }
  ]
}
```

此持久化至控制層的回傳對話紀錄歷史資料交換，位於 `Persistence Module` 將歷史對話紀錄回傳至後端控制層的流程位置。

相關流程定位方式可參考 [流程階段參考](./flow_stage_reference.md)。

## 補充說明

- 本文件描述的是 `Persistence Module` 至後端控制層的歷史對話紀錄資料交換
- 本文件先以單一打包完成的歷史對話紀錄資料作為最小資料物件
- 若後續需要展開回合資料或訊息資料的更細內容，再另行補充相關資料契約或資料實體草稿
