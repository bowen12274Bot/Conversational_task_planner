# 前端畫面需求

本文件用於整理 MVP 第一階段的前端畫面需求，重點在於支撐「輸入需求 -> 顯示文字回覆 -> 顯示初步規劃結果」的主流程。

---

## 頁面範圍

MVP 第一階段以前端單一主頁為核心。

現階段所有前端功能皆以支援此頁面的主流程展示為主，不先展開多頁面或複雜導航結構。

---

## 主畫面結構

主畫面至少應包含以下三個核心區塊：

- 需求輸入區
- 對話顯示區
- 規劃面板區

其中：

- 左欄以聊天對話窗為主
- 右欄以規劃面板為主

目前建議版面比例可先採：

- 左欄 35%
- 右欄 65%

文字排版示意如下：

```text
+----------------------------------------------------------------------------------+
|                                     主頁畫面                                     |
+----------------------------------+-----------------------------------------------+
|            聊天對話窗 35%         |                 規劃面板區 65%                 |
|                                  |                                               |
|  對話顯示區                       |  [主任務 A] ---- [主任務 B] ---- [主任務 C]     |
|  - 使用者輸入                     |    |                                          |
|  - 系統回覆                       |    +-- 子任務集合                              |
|  - 規劃安排 / 說明 / 原因          |        - 子任務 A-1                           |
|                                  |        - 子任務 A-2                            |
|                                  |    預估時間                                    |
|                                  |                                               |
|                                  |   ───────────────────────────────────────>    |
|                                  |   底部橫向時間軸 / 任務時間截線                 |
|                                  |                                               |
|----------------------------------|-----------------------------------------------|
|  輸入框                    送出   |                                               |
+----------------------------------+-----------------------------------------------+
```

---

## 視覺方向

整體風格以工具型、清楚、易讀為主。

可參考市面上常見 AI 對話工具的畫面語言，例如：

- ChatGPT
- Gemini
- Claude

但現階段的重點仍在於主流程是否可被清楚展示，而非高度還原特定產品風格。

---

## 各區塊需求

### 一、需求輸入區

需求輸入區應提供使用者輸入原始需求的主要入口。

此區塊至少應包含：

- 文字輸入欄位
- 送出操作

使用者輸入內容可為：

- 模糊需求
- 明確需求
- 較長的自然語言描述

### 二、對話顯示區

對話顯示區用於顯示系統與使用者之間的互動內容。

此區塊至少應能顯示：

- 使用者輸入內容
- 系統回覆文字

系統回覆文字現階段應至少能承接：

- 需求理解說明
- 追問內容
- 規劃安排與說明

### 三、規劃面板區

規劃面板區用於顯示目前已產生的初步規劃結果。

此區塊至少應能顯示：

- 主任務
- 對應主任務的子任務集合
- 任務塊下方的預估執行時間
- 底部橫向時間軸

規劃面板現階段的重點在於清楚呈現任務結構、工作層次與大致時序。

其呈現原則如下：

- 初始畫面先展示主任務架構
- 主任務依時間或相依關係排列於面板中
- 點開主任務後，再進入對應的子任務集合
- 規劃面板底部可有橫向時間軸
- 任務區塊中可有對應的時間截線，用於切分任務時間
- 任務塊下方可顯示預估執行時間

現階段先不談論編修能力。
